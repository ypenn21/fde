# GOOGLE-SPECIFIC TIPS

## What Google Looks For:
1. **Problem decomposition** - Break complex problems into smaller parts
2. **Communication** - Think out loud, explain trade-offs
3. **Code quality** - Clean, readable, handles edge cases
4. **Optimization** - Start simple, improve iteratively
5. **Testing mindset** - Verify your solution works

## Common Google Question Types:
- String manipulation with constraints
- Graph problems (especially BFS/DFS)
- Tree problems (especially BST operations)
- Dynamic programming (optimization, counting)
- System design elements (for FDE role)

## Edge Cases to ALWAYS Check:
- Empty input
- Single element
- All same elements
- Sorted/reverse sorted
- Negative numbers
- Integer overflow (use Python, no issue)
- Null/None values

## Red Flags to Avoid:
- Jumping into code without understanding the problem
- Not asking clarifying questions
- Silent coding
- Giving up when stuck (ask for hints!)
- Not testing your code
- Defensive when receiving feedback

# DAILY PRACTICE STRUCTURE

## Ideal Daily Session (3-4 hours):

### Warm-up (20 min)
- 1 Easy problem you've done before
- Focus on speed and clean code

### Main Practice (90-120 min)
- 2-3 new problems at your current level
- Take your time, understand deeply
- If stuck >20 min, look at hints (not solution)
- If stuck >30 min, study the solution, then re-implement

### Review (30 min)
- Add patterns to your notes
- Review what made you stuck
- Practice explaining the solution out loud

### Template Practice (20 min)
- Type out 2-3 templates from memory
- No looking at reference
